
## 2024-05-18 - Optimize DashboardScreen by Refactoring Local Methods to Const StatelessWidgets
**Learning:** In Flutter, using local functions (`_buildToolCard`, `_buildFileItem`) inside a `build` method to create UI components often leads to unnecessary rebuilds. By extracting these methods into their own `const StatelessWidget` classes (`_ToolCard`, `_FileItem`), the framework can skip rebuilding these specific components when their inputs (props) haven't changed, significantly reducing the overall rebuild time for the parent screen. In this specific application (`DashboardScreen`), refactoring reduced a 1000-iteration rebuild benchmark from ~16000ms down to ~10000ms, representing a nearly 37.5% performance improvement in the critical path of the main screen rendering.
**Action:** When working on Flutter performance optimization tasks, always look for large screens with multiple repetitive UI components generated via private local methods. Prioritize refactoring them into private `const StatelessWidget` classes within the same file to quickly yield measurable performance gains without sacrificing readability.
