## 2024-03-04 - [Performance Pattern: Unnecessary Object Allocation in build()]
**Learning:** Found a critical performance anti-pattern where a `TextEditingController` was being instantiated directly inside a `build()` method in `LawMapScreen` (`controller: TextEditingController(text: "IPC 302")`). Since `build()` can be called up to 60 times a second (e.g., during animations or scrolling), this was causing constant, unnecessary allocations of controller objects and, more importantly, a memory leak because the inline controller was never being disposed of.
**Action:** Always declare `TextEditingController` instances as state variables, initialize them in `initState()`, and ensure they are properly cleaned up in `dispose()` to prevent memory leaks and unnecessary garbage collection overhead during rapid rebuilds.
