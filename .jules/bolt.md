## 2024-05-24 - Extracting Flutter helper methods into StatelessWidgets
**Learning:** Refactoring local UI helper methods (like `_buildToolCard` or `_buildLawCard`) that return Widgets into explicit `const StatelessWidget` classes is a critical Flutter performance practice. It helps the framework's element tree diffing algorithm run more efficiently, better scopes rebuilds, and allows the engine to short-circuit the widget building process when const is used. Also discovered that `Colors.purple[50]` map lookups are not constant expressions, so explicit hex values like `Color(0xFFF3E5F5)` must be used for `const` constructors. To allow for `const` inputs, closures (e.g. `onTap`) capturing `context` must be replaced by static values, such as passing a `routeName` string.
**Action:** When auditing Flutter UI code for performance, actively look for private methods returning widgets and convert them to `const StatelessWidget` classes, ensuring all constructor parameters are static types and call sites are updated with the `const` keyword. Use hex color literals instead of `Colors.shade` to ensure widgets can be constructed as const.
