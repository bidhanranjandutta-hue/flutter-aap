## 2024-05-18 - Refactoring UI-building helper methods to `const StatelessWidget` classes
**Learning:** In Flutter, UI-building helper methods (e.g., `_buildWidget()`) execute during every parent rebuild. To optimize performance, we refactored these methods into dedicated private `StatelessWidget` classes and instantiate them with the `const` keyword. This allows the Flutter framework to completely skip the build phase for these sub-trees when the parent rebuilds. This is a common performance pattern for static UI components.
**Action:** When working on Flutter projects, actively look for static UI generation helper methods and convert them to `const StatelessWidget` classes. Also, replacing `Color.withOpacity(alpha)` with `Color.withValues(alpha: alpha)` prevents deprecation warnings and precision loss.
