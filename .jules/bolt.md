## 2026-04-05 - Encapsulate localized state toggles to prevent full screen rebuilds
**Learning:** When managing ephemeral local state (like segment toggles) in large Flutter screens, using `setState` causes the entire screen to rebuild. Encapsulating the toggle and its state into a separate `StatefulWidget` isolates the rebuild only to the affected UI node.
**Action:** Always extract localized UI controls into separate `StatefulWidget` classes rather than using inline `setState` or `ValueNotifier` when the state is exclusively used for a localized visual toggle.
## 2026-04-05 - ValueNotifier vs Full State Extraction
**Learning:** When encapsulating localized state controls (like segment toggles) into separate `StatefulWidget`s, it's critical to ensure the parent screen can still react to those changes if the state determines what content to display. A safer and equally performant approach for purely visual toggles that might later control parent content is to replace inline `setState` with `ValueNotifier` and `ValueListenableBuilder` within the existing widget. This isolates rebuilds without trapping the state in a child leaf node.
**Action:** Always prefer replacing `setState` with `ValueNotifier` and `ValueListenableBuilder` over extracting the entire toggle into a new `StatefulWidget`, unless the parent explicitly provides an `onChanged` callback to maintain the functional data flow.
