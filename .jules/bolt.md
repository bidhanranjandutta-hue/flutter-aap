## 2024-05-24 - Extracting Stateless Widgets from Helper Methods
**Learning:** Refactoring state-independent UI helper methods (like `_buildActionButton`) into `const StatelessWidget`s inside frequently rebuilding `StatefulWidget`s (like `OCRScannerScreen`) significantly reduces unnecessary widget tree rebuilds, saving layout and paint cycles. The true performance benefit comes from reducing widget tree rebuilds (saving layout and paint cycles) rather than micro-benchmarking object allocation.
**Action:** Always prefer creating private `const StatelessWidget` classes for UI components instead of using helper methods like `_buildWidget()` within a `StatefulWidget`'s State class, especially when the component doesn't depend on the mutable state. Cache InheritedWidget lookups like `Theme.of(context)` at the top of the `build` method.
