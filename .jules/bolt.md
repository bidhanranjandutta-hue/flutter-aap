## 2024-05-24 - Refactoring Local Helper Methods to const StatelessWidget for Performance
**Learning:** In Flutter, using local builder methods (e.g., `_buildToolCard`) within a `StatefulWidget` or large `StatelessWidget` evaluates the entire sub-tree during every rebuild. Extracting these methods into separate `const StatelessWidget` classes is a significant performance optimization as it allows the Flutter engine to short-circuit the widget building process using the `const` keyword and isolates rebuilds.
**Action:** When creating local helper methods that return Widgets, prefer extracting them into explicit `const StatelessWidget` classes. To ensure the `const` constructor can be used, pass static values (like explicit `Color(0xFF...)` instead of map lookups like `Colors.purple[50]!`) and strings for routing (instead of closures that capture `context`).
