## 2024-05-24 - Optimizing Flutter UI Rebuilds on Static Mockups
**Learning:** Extracting UI-building helper methods (e.g., `_buildLawCard`, `_buildLawHeader`) into dedicated private `const StatelessWidget` classes is a highly effective, low-risk way to optimize Flutter UI rebuilds on static mockups. This allows the Flutter framework to completely skip the build phase for these sub-trees when the parent rebuilds. This is often safer and more effective than attempting to isolate localized state with `ValueNotifier` on primarily static screens.
**Action:** Always prefer refactoring static helper methods into `const StatelessWidget` classes over complex state isolation when optimizing static mockups, ensuring to omit `super.key` to avoid lint warnings and prefix instantiations with `const`.
