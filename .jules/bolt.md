## 2024-05-24 - [Micro-benchmark allocation overhead]
**Learning:** [When benchmarking Flutter widgets to establish a performance baseline, micro-benchmarking allocation time often gets optimized away by the AOT compiler. The execution times inside automated WidgetTests for micro-optimizations (like caching `Theme.of(context)`) can fluctuate randomly based on system load, sometimes even appearing slower than the baseline (e.g. 4091ms vs 4221ms).]
**Action:** [Document the rationale that the true performance benefit comes from avoiding repeated InheritedWidget lookups traversing the map structure in `O(1)` constant time for every single usage within a complex `build` method. It prevents function call overhead. Pass this cached data directly to UI helper methods where applicable to maintain clean UI boundaries.]
