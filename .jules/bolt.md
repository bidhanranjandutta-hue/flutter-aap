## 2024-05-24 - Flutter UI-building helpers performance
**Learning:** Flutter UI-building helper methods (e.g., `_buildWidget()`) execute during every parent rebuild. Refactoring these methods into dedicated private `StatelessWidget` classes instantiated with the `const` keyword allows the Flutter framework to completely skip the build phase for these sub-trees when the parent rebuilds. This is an effective way to optimize primarily static mockups.
**Action:** When optimizing Flutter UI rebuilds on screens that are primarily static mockups, prioritize refactoring static `_buildHelper` methods into `const StatelessWidget` classes over attempting to isolate localized state with `ValueNotifier`. This safely optimizes build times without risking architectural state regressions.
