## 2024-05-24 - Extracting UI Helpers to Const StatelessWidgets
**Learning:** In Flutter, UI helper methods (like `_buildFeatureItem`) that return widgets without explicitly using the `const` keyword force the parent widget to reallocate and rebuild them entirely on every state change, consuming unnecessary layout and paint cycles. Furthermore, repeated `Theme.of(context)` lookups inside these helpers result in redundant O(depth) tree traversals.
**Action:** Extract static or stateless UI components into their own `const StatelessWidget` classes. Instantiate them using `const` in the parent widget to entirely remove them from the parent's rebuild cycle. Always cache InheritedWidget lookups (like `Theme.of(context)`) in a local variable at the top of the `build` method to mitigate tree traversal costs.
