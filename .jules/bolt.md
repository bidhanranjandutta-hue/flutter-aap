## 2024-05-24 - [Flutter Widget Rebuild Optimization]
**Learning:** Refactoring local UI helper methods (like `_buildFileItem` or `_buildToolCard`) that return Widgets into explicit `const StatelessWidget` classes is a recommended Flutter performance practice, allowing the Flutter engine to short-circuit the widget building process and avoid unnecessary rebuilds. This is specifically relevant in dynamic scrollable lists where helper methods would unnecessarily rebuild every frame, whereas a const widget will simply be re-used. Also learned `withOpacity` is deprecated and `withValues(alpha: ...)` should be used instead.
**Action:** Always extract static helper methods into `const StatelessWidget` in Flutter to improve performance and avoid using `withOpacity` by leveraging `withValues(alpha: ...)`.
